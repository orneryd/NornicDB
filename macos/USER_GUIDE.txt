1. INSTALLATION
================================================================================

REQUIREMENTS:
  - macOS 12.0 (Monterey) or later
  - ~500 MB disk space (1+ GB with AI models)

INSTALL STEPS:
  1. Download NornicDB-1.0.0-arm64.pkg
  2. Double-click the installer
  3. Follow the installation prompts
  4. Look for the NornicDB icon in your menu bar (top right of screen)

That's it! NornicDB is now installed and running.

================================================================================

6. SECURITY
================================================================================

CHANGE DEFAULT CREDENTIALS (IMPORTANT!)

Default credentials are:
  Username: admin
  Password: password

To change:
  1. Open Settings (âŒ˜,)
  2. Go to Security tab
  3. Update Username and Password
  4. Click "Generate Random Secret" for JWT
  5. Click "Save & Restart Server"

ENABLE DATABASE ENCRYPTION

To encrypt data at rest:
  1. Open Settings â†’ Security tab
  2. Check "Enable Encryption at Rest"
  3. Click "Generate Strong Key" (or enter your own 16+ character key)
  4. SAVE THIS KEY SECURELY - data cannot be recovered without it!
  5. Click "Save & Restart Server"

âš ï¸  WARNING: If you lose your encryption key, your data is permanently lost!

GENERATE API TOKENS (For MCP Servers, Automation)

  1. Login to Web UI: http://localhost:7474
  2. Navigate to: /security page
  3. Enter Token Subject (e.g., "MCP Server")
  4. Choose Expiry (1 hour, 1 day, 1 week, 30 days, never)
  5. Click "Generate Token"
  6. Copy token immediately (it won't be shown again)
  7. Use in Authorization header: Bearer <token>

7. COMMAND LINE USAGE
================================================================================

NornicDB binary is installed at: /usr/local/bin/nornicdb

BASIC COMMANDS:

  Start server:
    $ nornicdb serve

  Start with custom config:
    $ nornicdb serve --config ~/.nornicdb/config.yaml

  Start with custom ports:
    $ nornicdb serve --bolt-port 7687 --http-port 7474

  Check version:
    $ nornicdb version

  View help:
    $ nornicdb --help

SERVICE MANAGEMENT (Advanced):

  Check if service is running:
    $ launchctl list | grep nornicdb

  Start service manually:
    $ launchctl start com.nornicdb.server

  Stop service manually:
    $ launchctl stop com.nornicdb.server

  Restart service:
    $ launchctl kickstart -k gui/$(id -u)/com.nornicdb.server

  View logs:
    $ tail -f /usr/local/var/log/nornicdb/stdout.log
    $ tail -f /usr/local/var/log/nornicdb/stderr.log

CONFIGURATION FILE:

  Location: ~/.nornicdb/config.yaml
  
  Edit with:
    $ nano ~/.nornicdb/config.yaml
    
  View current config:
    $ cat ~/.nornicdb/config.yaml

DIRECTORY STRUCTURE:

  /usr/local/bin/nornicdb              Binary
  /usr/local/var/nornicdb/data/        Database files
  /usr/local/var/nornicdb/models/      AI models (.gguf)
  /usr/local/var/log/nornicdb/         Log files
  /Applications/NornicDB.app           Menu bar app
  ~/.nornicdb/config.yaml              Configuration
  ~/Library/LaunchAgents/              Auto-start configs
